States are handled by the \doxysectlink{state}{State}{0} class. State objects are organised in dicts, where keys are represented by tuples of the form $ (n, \ell) $ with values representing subsystem density matrices. Thereby, $ n $ indicates the center of the corresponding subsystem and $ \ell $ represents its scale (i.\+e. the extend starting from 0 for a subsystem containing a single spin). LITE can evolve any State object.

LITE involves a bunch of parameters for integrating the Schr√∂dinger/\+Lindblad equation, the minimization for information and the algorithm itself. A documentation of all parameters can be found \doxysectlink{md__2_users_2christoph_2local-information_2local__information_2docs_2config}{here}{0}. Algorithmically, the most important parameters are \$ min\+\_\+l \$ and \$ max\+\_\+l \$, where \$ max\+\_\+l \$ sets the largest scale at which time evolution of subsystems is performed before the information is minimized at scale \$ min\+\_\+l \$. Default values are {\ttfamily min\+\_\+l = 3} and {\ttfamily max\+\_\+l = 5} (typically larger values are required for high quality results). \hypertarget{index_autotoc_md29}{}\doxysubsection{\texorpdfstring{Time\+Evolution\+Config}{Time\+Evolution\+Config}}\label{index_autotoc_md29}

 Another parameter worth to be mentioned here (since it can potentially strongly influence the outcome of simulations) is {\ttfamily minimization\+\_\+threshold} controlling the frequency of information minimization. By default it is set to $ 1\times 10^{-2}$ and (from our empirical experience) should not be increased beyond $ 5\times 10^{-2}$ or dropped below $ 1\times 10^{-3}$.
